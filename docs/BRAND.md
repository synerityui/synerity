# Synerity — Brand Guide
Version 1.0 · May 2026

---

## The Name

**Library:** Synerity (published as `@synerity/ui`)
**Company:** Synerity Labs
**Legal entity:** Synerity Labs Private Limited
**GitHub org:** github.com/synerityui
**npm scope:** @synerity (owned)

Synerity is a coined word — a blend of *synergy* (working together) and *clarity* (clear, no friction). It signals a library where components work together without drama.

---

## Tagline

**Primary:** Headless by design. Beautiful by default.

**Alternatives:**
- Components that work together.
- Zero runtime. Full design freedom.
- Build beautiful interfaces, piece by piece.

---

## npm Package Architecture

| Package | Purpose |
|---|---|
| @synerity/ui | Styled components, default design system skin |
| @synerity/headless | Logic + a11y layer, zero CSS — use standalone |
| @synerity/tokens | Design tokens, CSS custom properties |
| @synerity/icons | Icon set, fully tree-shakeable |
| @synerity/forms | Form primitives with built-in validation |
| @synerity/cli | Copy-paste installer (npx synerity add button) |

## Domain

**Primary (recommended):** synerity.dev
**Fallback:** synerity.io or syneritylabs.dev

---

## Positioning

Synerity is the first React UI library combining:
- Zero-runtime CSS (CSS Modules, no Emotion)
- RSC / React Server Components compatibility
- WCAG 2.1 AA verified accessibility
- Strict TypeScript (no `any`)
- Design token theming via CSS custom properties
- Stable APIs with codemods (no painful major migrations)
- Both npm and copy-paste distribution
